üîó Multiple Conditions with AND/OR

Sometimes you need to filter data with multiple conditions:

SELECT * FROM products WHERE price > 500 AND category = 'Phone'; -- Both conditions must be TRUE
SELECT * FROM products WHERE category = 'Phone' OR category = 'Tablet'; -- Either condition can be TRUE

üß† Understanding AND vs OR

Operator Logic Example Result
AND Both must be true price > 500 AND rating >= 4.5 Expensive AND highly rated
OR Either can be true category = 'Phone' OR category = 'Tablet' Phones OR tablets
Pro Tip: Use parentheses () to group conditions when mixing AND/OR!

üìä Sorting Results with ORDER BY

Control the order of your results:

SELECT * FROM products ORDER BY price ASC; -- Sort by price: lowest to highest
SELECT * FROM products ORDER BY rating DESC; -- Sort by rating: highest to lowest

üîÑ ASC vs DESC

Direction Full Name Numbers Text Dates
ASC Ascending 1, 2, 3, 4, 5 A, B, C, D, E Oldest to Newest
DESC Descending 5, 4, 3, 2, 1 Z, Y, X, W, V Newest to Oldest
Default: If you don't specify ASC or DESC, SQL uses ASC (ascending) by default.

üéØ Limiting Results with LIMIT

Get only the top N results:

SELECT * FROM products ORDER BY price DESC LIMIT 3; -- Get top 3 most expensive products
SELECT * FROM products ORDER BY rating DESC LIMIT 5; -- Get top 5 highest rated products

üí° Why Use LIMIT?

Use Case Example Business Value
Top performers LIMIT 10 Best selling products
Recent data ORDER BY date DESC LIMIT 5 Latest orders
Performance LIMIT 100 Faster queries on large tables
